## 方案 A 实现：Gateway WebSocket 直连 (09:20)

### 架构变更

**原架构**：
```
前端 → 后端 SSE → OpenAI 兼容 API → Gateway → AI
```
- 问题：工具定义无法传递给 AI

**新架构**：
```
前端 → Gateway WebSocket → AI（工具在 Gateway 层执行）
```
- 优势：获得完整的工具支持

### 新增文件

| 文件 | 说明 |
|------|------|
| `src/services/gateway-ws.ts` | Gateway WebSocket 客户端 |
| `src/services/chat-gateway.ts` | 聊天服务（支持 Gateway/SSE 双模式） |
| `.env` | 环境变量配置 |

### Gateway WebSocket 配置

```env
VITE_GATEWAY_WS_URL=ws://127.0.0.1:18789
VITE_GATEWAY_TOKEN=77784ff3d8dec1f92514d785195227cdfda0b15867416fd0
```

### Gateway RPC 方法

| 方法 | 说明 |
|------|------|
| `connect` | 连接认证 |
| `sessions.list` | 获取会话列表 |
| `sessions.spawn` | 创建会话 |
| `chat.send` | 发送消息（非阻塞） |
| `chat.history` | 获取聊天历史 |
| `chat.abort` | 中止运行 |

### 事件类型

| 事件 | 说明 |
|------|------|
| `chat` | 聊天消息（流式） |
| `tool_call` | 工具调用开始 |
| `tool_result` | 工具执行结果 |
| `session.started` | 会话开始 |
| `session.ended` | 会话结束 |

### 双模式支持

聊天服务支持自动切换：
1. **优先**：Gateway WebSocket（支持完整工具）
2. **备用**：后端 SSE（基础对话）

```typescript
if (gatewayWS.isConnected()) {
  // 使用 Gateway，获得工具支持
} else {
  // 回退到 SSE
}
```

## TaskMonitor 数据源修复 (10:47)

### 问题
TaskMonitor 组件使用前端本地 `taskScheduler`（localStorage），与后端数据库任务系统完全独立。

### 原来的数据流
```
前端 localStorage → TaskMonitor 显示
(与后端数据库无关)
```

### 修复后
```
后端 PostgreSQL → /api/tasks → TaskMonitor 显示
(真实任务数据)
```

### 改动
- `TaskMonitor.tsx` 重写，从 `/api/tasks` 获取真实数据
- 移除对 `taskScheduler` 的依赖
- 添加完成任务、删除任务操作
- 显示 "PostgreSQL 实时数据" 标识

### 两套任务系统说明

| 系统 | 用途 | 存储 |
|------|------|------|
| `/api/tasks` | 用户创建的任务 | PostgreSQL |
| `taskScheduler` | 后台队列任务 (agent_chat, code_generation) | localStorage |

用户任务监控显示的是 `/api/tasks` 数据。
