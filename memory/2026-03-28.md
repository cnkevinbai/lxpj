### Session #7 - 系统稳定性与可观测性 + 故障诊断模式

**学习时段**: 凌晨 2:30-4:00

#### 学习成果

**1. 可观测性三大支柱**
- Logs (日志)：离散事件，问题定位
- Metrics (指标)：聚合数值，趋势分析
- Traces (追踪)：请求路径，分布式调用链

**2. iov-platform 可观测性实现分析**
- HealthStatus 健康状态枚举 (5 种状态)
- HealthCheckResult 健康检查结果类
- MetricsRegistry 指标注册表 (ConcurrentHashMap)
- MetricsExporter Prometheus 导出器 (HTTP 9090/metrics)
- CircuitBreakerImpl 熔断器实现 (AtomicReference 状态切换)

**3. 熔断器系统详解**
- 状态机：CLOSED → OPEN → HALF_OPEN
- 配置：failureRateThreshold (50%), waitDuration (60s)
- 预设：strict(), lenient(), defaults()
- 无锁设计：AtomicReference + AtomicInteger

**4. SRE 核心概念**
- SLI/SLO/SLA 框架
- 错误预算计算 (99.9% SLO = 8.76h/year)
- 故障诊断五步法
- 5 Whys 根因分析法

**5. 系统稳定性模式**
- 熔断器：快速失败，避免级联故障
- 限流器：保护系统不过载 (待实现)
- 重试机制：处理临时故障 (待实现)
- 降级策略：保证核心功能 (待实现)

#### 关键洞察

1. **可观测性 ≠ 监控**
   - 监控：已知问题检测
   - 可观测性：未知问题诊断

2. **统一 trace_id 是关键**
   - Logs/Metrics/Traces 三者关联
   - 完整诊断链路

3. **错误预算驱动决策**
   - 预算充足 → 发布新功能
   - 预算耗尽 → 专注稳定性

## 后续优化工作

### 编译问题修复

**问题**: gateway 模块 JwtUtil.java 编译错误 - 找不到符号 `log`

**原因**: 父 pom.xml 的 `maven-compiler-plugin` 缺少 Lombok 注解处理器配置

**修复**: 在父 pom.xml 添加 annotationProcessorPaths 配置
```xml
<annotationProcessorPaths>
    <path>
        <groupId>org.projectlombok</groupId>
        <artifactId>lombok</artifactId>
        <version>1.18.30</version>
    </path>
</annotationProcessorPaths>
```

**验证**: 全项目 `mvn compile` 成功 ✅
