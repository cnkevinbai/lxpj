## 技术要点

### 后端
1. **鉴权码设计**: 时间戳(8位) + 随机数(8位) + 校验位(4位)
2. **遗嘱消息**: QoS=1, Retained=true, 自动发布离线状态
3. **签名算法**: HMAC-SHA256(secretKey, terminalId + timestamp + nonce)
4. **幂等处理**: Nonce 缓存防重放，时间戳有效期 5 分钟
5. **设备影子**: reported(设备上报) + desired(平台下发) 双向同步

### 前端
1. **类型系统**: 完整的 TypeScript 类型定义
2. **状态管理**: Zustand 实现，支持实时更新
3. **组件化**: 可复用的绑定相关组件
4. **三种协议**: 协议差异展示，流程说明

### 安全合规
1. **SM4 加密**: 128位密钥，ECB/CBC模式
2. **SM3 哈希**: 256位输出，密码存储
3. **数据脱敏**: 8种敏感数据类型支持
4. **JWT 认证**: Token 验证、用户信息传递

### 网关
1. **路由转发**: 11个服务路由配置
2. **限流熔断**: Redis 限流 + Resilience4j 熔断
3. **认证鉴权**: JWT Token 验证
4. **降级处理**: 服务不可用时友好响应
