## 🎯 核心收获

### 一、数学与算法直觉

**关键认知**: 算法直觉不是天赋，是可训练的技能

**训练框架**:
```
算法直觉 = 模式识别 + 复杂度感知 + 抽象建模

训练方法:
1. 可视化学习 — 3Blue1Brown 几何直觉理解
2. 对比分析 — 同一问题多解法对比 (O(n²) vs O(n log n) vs O(n))
3. 第一性原理 — 从问题本质推导，不记忆步骤
```

**每日训练计划**:
- LeetCode 1 题 (30 分钟)
- 5 分钟分类 → 10 分钟暴力 → 10 分钟优化 → 5 分钟反思
- 每周总结模式识别进展

### 二、成本意识

**关键认知**: 成本意识是工程师的核心竞争力

**评估框架**:
```
ROI 优先级矩阵:
        高价值
          ↑
   P1     │     P0
(低成本低价值)│(高成本低价值) ← 避免
──────────┼──────────
   P3     │     P2
(低成本低价值)│(高成本低价值) ← 优先
          │
        低成本 → 高成本
```

**决策流程**:
1. 量化价值 (影响用户数/效率提升)
2. 估算成本 (开发 + 维护 + 风险)
3. 计算 ROI = 价值/成本
4. 对比替代方案 (80% 效果的更便宜方法)
5. 决定做不做 + 何时做

### 三、系统设计权衡

**核心洞察**: 一切皆 trade-off，没有最优解

**关键权衡**:
- 延迟 vs 吞吐
- 一致性 vs 可用性 (CAP 定理)
- 成本 vs 性能

**关键问题**:
- 我们真的需要这个级别的性能吗？
- 用户能感知 100ms 和 50ms 的区别吗？
- 这个优化值得花 3 天吗？
