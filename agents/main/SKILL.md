## 📋 元技能详情

### 1. 任务理解

**触发**: 收到用户消息
**输出**: 任务类型、复杂度、意图

**关键能力**:
- 意图识别（创建/修改/查询/分析）
- 任务类型分类（代码/设计/测试/运维）
- 复杂度评估（简单/中等/复杂）
- 可并行性判断

**详细文档**: [task-analysis.md](./skills/task-analysis.md)

---

### 2. 任务路由

**触发**: 任务分析完成
**输出**: Agent + 模型 + 技能

**路由矩阵**:
```
代码任务 → qwen3-coder-next → frontend-dev/backend-dev
架构任务 → qwen3-max → architect
产品任务 → glm-5 → product-manager
设计任务 → glm-5 → ui-ux-designer
测试任务 → qwen3.5-plus → test-engineer
安全任务 → qwen3-max → security-engineer
运维任务 → qwen3.5-plus → devops-engineer
文档任务 → kimi-k2.5 → coordinator
审查任务 → qwen3-max → code-reviewer
```

**详细文档**: [task-routing.md](./skills/task-routing.md)

---

### 3. 任务分解

**触发**: 复杂任务、多模块任务
**输出**: 子任务列表 + 依赖关系

**分解模式**:
- 顺序依赖: T1 → T2 → T3
- 并行独立: T1, T2, T3 同时执行
- 混合模式: 部分并行，部分顺序

**详细文档**: [task-decomposition.md](./skills/task-decomposition.md)

---

### 4. 结果聚合

**触发**: 子 Agent 完成
**输出**: 整合后的最终结果

**聚合策略**:
- 代码聚合: 合并文件、依赖
- 报告聚合: 整合章节
- 分析聚合: 汇总发现和建议

**详细文档**: [result-aggregation.md](./skills/result-aggregation.md)

---

### 5. 错误恢复

**触发**: 任务执行失败
**输出**: 恢复策略执行结果

**恢复策略**:
- 重试: 最多 2 次，指数退避
- 降级: 换模型/换 Agent
- 回滚: 撤销已执行步骤

**详细文档**: [error-recovery.md](./skills/error-recovery.md)

---

### 6. 上下文管理

**触发**: 会话开始/持续
**输出**: 状态维护、历史追踪

**管理内容**:
- 会话状态
- 任务历史
- 用户偏好

**详细文档**: [context-management.md](./skills/context-management.md)
