# 系统设计技能

## 📋 技能说明

设计高可用、可扩展、可维护的系统架构，输出完整的架构设计文档。

---

## 🎯 适用场景

- 新系统架构设计
- 系统重构规划
- 技术方案评审
- 架构决策记录

---

## 3. 数据模型

### 3.1 核心实体

```prisma
model User {
  id        String   @id @default(uuid())
  email     String   @unique
  name      String
  role      UserRole @default(USER)
  status    UserStatus @default(ACTIVE)
  createdAt DateTime @default(now())
  updatedAt DateTime @updatedAt
  
  orders    Order[]
  
  @@index([email])
  @@map("users")
}
```

## 4. 接口设计

### 4.1 API 规范

- 基础路径：`/api/v1`
- 认证方式：Bearer JWT
- 响应格式：JSON

### 4.2 核心接口

| 接口 | 方法 | 说明 |
|-----|------|------|
| /users | GET | 获取用户列表 |
| /users/:id | GET | 获取用户详情 |
| /users | POST | 创建用户 |
| /users/:id | PUT | 更新用户 |

## 5. 非功能性需求

### 5.1 性能要求

| 指标 | 目标值 |
|-----|--------|
| API 响应时间 | < 200ms (P95) |
| 并发用户数 | > 1000 |
| QPS | > 5000 |

### 5.2 可用性要求

| 指标 | 目标值 |
|-----|--------|
| 可用性 | > 99.9% |
| 故障恢复时间 | < 5min |
| 数据备份 | 每日增量 + 全量 |

### 5.3 安全要求

- [ ] HTTPS 强制
- [ ] API 限流
- [ ] SQL 注入防护
- [ ] XSS 防护
- [ ] CSRF 防护

## 6. 部署架构

### 6.1 环境规划

| 环境 | 用途 | 配置 |
|-----|------|------|
| 开发 | 开发调试 | 单机部署 |
| 测试 | 功能测试 | 单机部署 |
| 预发 | 预发布验证 | 集群部署 |
| 生产 | 正式运行 | 集群部署 |

### 6.2 CI/CD 流程

```
代码提交 → 单元测试 → 构建 → 部署测试 → 部署生产
```

## 7. 风险与缓解

| 风险 | 影响 | 概率 | 缓解措施 |
|-----|------|------|---------|
| 数据库瓶颈 | 高 | 中 | 读写分离、分库分表 |
| 单点故障 | 高 | 低 | 集群部署、故障转移 |
| 安全漏洞 | 高 | 低 | 定期安全审计 |

## 8. 附录

### 8.1 术语表

| 术语 | 说明 |
|-----|------|
| QPS | 每秒请求数 |
| P95 | 95% 请求的响应时间 |
