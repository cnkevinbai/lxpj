# PRD 文档编写技能

## 📋 技能说明

编写清晰、完整的产品需求文档（PRD）。

---

## 🎯 适用场景

- 产品需求文档编写
- 功能规格说明
- 产品规划

---

## 📝 PRD 模板

```markdown
# 产品需求文档：[产品名称]

## 1. 文档信息
| 项目 | 内容 |
|-----|------|
| 版本 | v1.0 |
| 作者 | xxx |
| 日期 | 2026-03-16 |
| 状态 | 草稿/评审/通过 |

## 2. 背景与目标
### 2.1 背景
[为什么需要这个产品]

### 2.2 目标
- 目标1
- 目标2

## 3. 用户范围
| 用户类型 | 描述 | 核心诉求 |
|---------|------|---------|

## 4. 功能需求
### 4.1 功能列表
| 功能 | 优先级 | 状态 |
|-----|--------|------|

### 4.2 功能详情
#### 4.2.1 [功能名称]
- 功能描述
- 业务流程
- 界面原型
- 接口说明

## 5. 非功能需求
### 5.1 性能需求
### 5.2 安全需求
### 5.3 兼容性需求

## 6. 附录
```

---

## ✅ 检查清单

- [ ] 背景目标清晰
- [ ] 功能列表完整
- [ ] 有界面原型
- [ ] 验收标准明确
