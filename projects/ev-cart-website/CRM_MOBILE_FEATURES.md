## 🔍 查重功能

### 1. 线索查重 ✅

**API**: `GET /api/v1/leads/duplicate-check`

**参数**:
| 参数 | 类型 | 说明 |
|-----|------|------|
| phone | string | 手机号 (必填) |
| days | number | 查重天数 (默认 30 天) |

**响应**:
```json
{
  "isDuplicate": true,
  "count": 2,
  "existingLeads": [
    {
      "id": "uuid",
      "name": "张三",
      "status": "new",
      "createdAt": "2026-03-11T10:00:00.000Z",
      "ownerId": "sales_id"
    }
  ]
}
```

**查重逻辑**:
1. 输入手机号时自动触发查重
2. 查找 30 天内的重复线索
3. 显示重复线索列表
4. 提示用户确认是否继续提交

---

### 2. 客户查重 ✅

**API**: `GET /api/v1/customers/duplicate-check`

**参数**:
| 参数 | 类型 | 说明 |
|-----|------|------|
| phone | string | 联系电话 |
| email | string | 联系邮箱 |

**响应**:
```json
{
  "isDuplicate": true,
  "count": 1,
  "existingCustomers": [
    {
      "id": "uuid",
      "name": "某某公司",
      "contactPhone": "13800138000",
      "contactEmail": "contact@example.com",
      "status": "active",
      "createdAt": "2026-03-11T10:00:00.000Z"
    }
  ]
}
```

**查重逻辑**:
1. 输入手机号或邮箱时自动触发查重
2. 查找所有重复客户
3. 显示重复客户列表
4. 提示用户确认是否继续提交

## 📊 使用场景

### 场景 1：展会现场录入

1. 业务员在展会现场遇到潜在客户
2. 打开 CRM 移动端 `/crm/mobile`
3. 点击「录入线索」
4. 填写客户信息
5. 系统自动查重
6. 确认提交

### 场景 2：拜访客户录入

1. 业务员拜访客户
2. 扫描客户名片 (预留扫码功能)
3. 自动填充信息
4. 系统自动查重
5. 确认提交

### 场景 3：电话销售录入

1. 业务员接到咨询电话
2. 打开 CRM 移动端
3. 录入客户信息
4. 系统自动查重
5. 如有重复，显示现有客户
6. 确认提交或关联现有客户
