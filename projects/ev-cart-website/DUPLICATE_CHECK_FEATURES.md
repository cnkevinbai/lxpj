## 🔍 智能查重算法

### 多维度匹配

| 维度 | 权重 | 说明 |
|-----|------|------|
| 手机号 | 50% | 精确匹配 |
| 邮箱 | 40% | 精确匹配 |
| 公司名 | 30% | 模糊匹配 |

### 匹配度计算

```
匹配度 = 手机号匹配 (50%) + 邮箱匹配 (40%) + 公司名匹配 (30%)
```

### 查重建议

| 匹配度 | 建议 | 说明 |
|-------|------|------|
| ≥80% | 合并 (merge) | 高度相似，建议合并 |
| 50%-79% | 关联 (link) | 中度相似，建议关联 |
| <50% | 创建 (create) | 低相似，可以创建 |

## 📝 实际应用场景

### 场景 1：展会现场录入

**流程**:
1. 业务员在展会收集名片
2. 打开 CRM 移动端录入线索
3. 输入手机号，系统自动查重
4. 发现 30 天内已有相同线索
5. 系统提示"发现重复线索，建议关联"
6. 业务员选择关联现有线索
7. 新线索自动关联到现有线索

**效果**:
- 避免重复跟进
- 提高销售效率
- 改善客户体验

## 🔌 API 接口

### 线索查重

```typescript
// 请求
GET /api/v1/leads/duplicate-check
Params:
  - phone?: string
  - email?: string
  - company?: string
  - days?: number (default: 30)

// 响应
{
  "isDuplicate": true,
  "count": 2,
  "score": 90,
  "existingLeads": [
    {
      "id": "uuid",
      "name": "张三",
      "phone": "13800138000",
      "matchFields": ["phone", "email"],
      "matchScore": 90,
      "ownerName": "销售员 A"
    }
  ],
  "suggestion": "merge"
}
```

### 客户查重

```typescript
// 请求
GET /api/v1/customers/duplicate-check
Params:
  - phone?: string
  - email?: string
  - company?: string

// 响应
{
  "isDuplicate": true,
  "count": 1,
  "score": 50,
  "existingCustomers": [
    {
      "id": "uuid",
      "name": "某某公司",
      "contactPhone": "13800138000",
      "matchFields": ["phone"],
      "matchScore": 50
    }
  ],
  "suggestion": "link"
}
```

### 合并线索

```typescript
// 请求
POST /api/v1/leads/merge
Body:
{
  "targetId": "uuid",
  "sourceIds": ["uuid1", "uuid2"]
}

// 响应
{
  "success": true,
  "data": { ...mergedLead }
}
```

### 关联线索

```typescript
// 请求
POST /api/v1/leads/link
Body:
{
  "leadId": "uuid",
  "relatedLeadIds": ["uuid1", "uuid2"]
}

// 响应
{
  "success": true,
  "leadId": "uuid",
  "relatedLeadIds": ["uuid1", "uuid2"]
}
```

### 查重统计

```typescript
// 请求
GET /api/v1/leads/duplicate-stats
Params:
  - days?: number (default: 30)

// 响应
{
  "totalLeads": 1000,
  "duplicateCount": 50,
  "duplicateRate": "5.00%",
  "topDuplicates": [
    {
      "phone": "13800138000",
      "count": 3
    }
  ]
}
```
