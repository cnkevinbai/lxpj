## 👥 角色权限分配

### 售后主管 (service_manager)

**核心职责**: 工单分配、团队管理

**权限**:
| 权限 | 状态 | 说明 |
|-----|------|------|
| 查看工单 | ✅ | 查看所有工单 |
| 创建工单 | ✅ | 创建工单 |
| **分配工单** | ✅ | **核心权限** - 分配给技术人员 |
| 受理工单 | ✅ | 受理工单 |
| 处理工单 | ❌ | 不直接处理 |
| 完成工单 | ❌ | 不直接完成 |
| 关闭工单 | ✅ | 关闭工单 |
| 导出工单 | ✅ | 导出数据 |
| 查看统计 | ✅ | 查看团队统计 |

**数据权限**: 本部门全部数据

---

### 服务人员 (technician)

**核心职责**: 上门服务、工单处理

**权限**:
| 权限 | 状态 | 说明 |
|-----|------|------|
| 查看工单 | ✅ | 查看分配给自己的工单 |
| 创建工单 | ❌ | 不创建工单 |
| 分配工单 | ❌ | 不分配工单 |
| 受理工单 | ❌ | 不受理工单 |
| **处理工单** | ✅ | **核心权限** - 处理工单 |
| **完成工单** | ✅ | **核心权限** - 完成服务 |
| 关闭工单 | ❌ | 不能关闭 (客户确认后关闭) |
| 导出工单 | ❌ | 不导出数据 |
| 查看统计 | ❌ | 不查看统计 |

**数据权限**: 仅自己的工单

| 查看统计 | ✅ | 查看统计 |

## 🔄 工单分配流程

### 主管分配场景

```
┌─────────────────────────────────────────────────────────┐
│                    售后主管                              │
│                                                         │
│  1. 查看待分配工单列表                                   │
│     - 按优先级排序 (特急 > 紧急 > 普通)                  │
│     - 按创建时间排序                                    │
│                                                         │
│  2. 选择工单 (支持批量)                                 │
│     - 单选                                              │
│     - 多选/批量                                         │
│                                                         │
│  3. 选择服务人员                                        │
│     - 查看技术人员列表                                  │
│     - 查看技术人员技能                                  │
│     - 查看技术人员评分                                  │
│     - 查看技术人员当前工单数                            │
│                                                         │
│  4. 确认分配                                            │
│     - 系统自动通知技术人员                              │
│     - 系统自动通知客户 (短信/APP 推送)                    │
│     - 记录分配人 (主管)                                 │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

### 工单状态流转

```
客户申请
    ↓
待受理 (pending)
    ↓
已受理 (accepted) ← 客服受理
    ↓
已分配 (assigned) ← 主管分配 ★
    ↓
处理中 (processing) ← 技术人员处理
    ↓
待确认 (waiting_confirm) ← 技术人员完成
    ↓
已关闭 (closed) ← 客户确认
```

## 🎯 新增 API 接口

### 工单分配相关

| 方法 | 路径 | 说明 | 权限 |
|-----|------|------|------|
| GET | `/after-sales/tickets/to-assign` | 获取待分配工单 | service_manager |
| POST | `/after-sales/tickets/:id/assign` | 分配单个工单 | service_manager |
| POST | `/after-sales/tickets/batch-assign` | 批量分配工单 | service_manager |
| GET | `/after-sales/tickets/my` | 获取我的工单 | technician |

### 角色权限相关

| 方法 | 路径 | 说明 |
|-----|------|------|
| GET | `/rbac/roles` | 获取所有角色 |
| GET | `/rbac/permissions` | 获取所有权限 |
| GET | `/rbac/users/:id/roles` | 获取用户角色 |
| POST | `/rbac/users/:id/roles` | 分配角色 |
| GET | `/rbac/users/:id/permissions` | 获取用户权限 |

## 📊 数据权限设计

### 数据范围

| 角色 | 数据范围 | 说明 |
|-----|---------|------|
| **系统管理员** | 全部数据 | 所有部门所有数据 |
| **售后主管** | 本部门全部 | 服务部门所有工单 |
| **客服人员** | 本部门全部 | 服务部门所有工单 |
| **服务人员** | 仅自己 | 只查看自己的工单 |

### 数据过滤

```typescript
// 服务主管查询
WHERE department = 'service'

// 服务人员查询
WHERE technician_id = :currentUserId
```
