## 🔧 技术实现方案

### API 接口设计

#### 客户相关
```typescript
// 官网→CRM
POST /api/v1/website/customers      // 客户注册
POST /api/v1/website/leads          // 线索创建
POST /api/v1/website/opportunities  // 商机创建
POST /api/v1/website/tickets        // 工单创建

// CRM→官网
GET  /api/v1/website/customers/:id  // 客户信息
GET  /api/v1/website/orders         // 订单列表
GET  /api/v1/website/tickets        // 工单列表
```

#### 产品相关
```typescript
// CRM→官网
GET  /api/v1/website/products       // 产品列表
GET  /api/v1/website/products/:id   // 产品详情
GET  /api/v1/website/inventory      // 库存查询
```

#### 内容相关
```typescript
// CRM→官网
GET  /api/v1/website/news           // 新闻列表
GET  /api/v1/website/cases          // 案例列表
GET  /api/v1/website/solutions      // 方案列表
```

### 数据同步策略

| 数据类型 | 同步方式 | 频率 |
|---------|---------|------|
| 产品信息 | API 拉取 | 实时 |
| 产品价格 | API 拉取 | 实时 |
| 库存数量 | API 拉取 | 5 分钟 |
| 新闻内容 | API 拉取 | 10 分钟 |
| 案例内容 | API 拉取 | 10 分钟 |
| 客户数据 | API 推送 | 实时 |
| 线索数据 | API 推送 | 实时 |

### 缓存策略

```typescript
// Redis 缓存配置
{
  products: { ttl: 300 },      // 产品 5 分钟
  news: { ttl: 600 },          // 新闻 10 分钟
  cases: { ttl: 600 },         // 案例 10 分钟
  inventory: { ttl: 60 },      // 库存 1 分钟
  customer: { ttl: 0 }         // 客户不缓存
}
```

## 🔒 安全策略

### 认证授权
- 官网公开内容：无需认证
- 客户门户：JWT Token 认证
- CRM 后台：JWT Token + 角色权限

### 数据保护
- 敏感数据加密传输（HTTPS）
- 密码加密存储（bcrypt）
- 个人信息脱敏展示

### 防攻击策略
- API 限流（100 次/分钟/IP）
- 验证码（注册/登录/提交表单）
- SQL 注入防护（TypeORM）
- XSS 防护（输入过滤）
