## 📦 模块详解

### 1. 服务工单管理 (Service Ticket Management)

**路径**: `/service-tickets`  
**后端**: `backend/src/modules/after-sales/`

#### 核心功能
| 功能 | 说明 | 状态 |
|------|------|------|
| 工单创建 | 客户报修/客服创建/系统自动生成 | ✅ |
| 工单分配 | 自动分配/手动派单 | ✅ |
| 工单处理 | 工程师接单、处理、反馈 | ✅ |
| 工单关闭 | 客户确认、工单关闭 | ✅ |
| 工单升级 | 复杂问题升级处理 | ✅ |
| 工单统计 | 工单量、处理时效统计 | ✅ |

#### 工单类型
| 类型 | 说明 | 优先级 |
|------|------|--------|
| 安装调试 | 产品安装、调试 | 中 |
| 维修维护 | 产品维修、保养 | 高 |
| 技术咨询 | 技术问题咨询 | 低 |
| 投诉建议 | 客户投诉、建议 | 高 |
| 退换货 | 产品退换 | 中 |

#### 工单优先级
| 优先级 | 响应时间 | 处理时间 | 说明 |
|--------|----------|----------|------|
| P0-紧急 | 15 分钟 | 4 小时 | 生产停机、重大故障 |
| P1-高 | 1 小时 | 24 小时 | 功能受影响 |
| P2-中 | 4 小时 | 3 天 | 一般问题 |
| P3-低 | 24 小时 | 7 天 | 咨询、建议 |

#### 工单状态流转
```
新建 → 已分配 → 已接单 → 处理中 → 待确认 → 已完成
              ↓           ↓
           已超时      客户不满意
              ↓           ↓
           升级处理    重新处理
```

## 🔄 核心业务流程

### 服务工单流程
```
客户报修 → 工单创建 → 工单分配 → 工程师接单 → 联系客户
    ↓
上门服务 → 问题处理 → 服务完成 → 客户确认 → 客户评价
```

### 配件领用流程
```
工程师申请 → 主管审批 → 仓库发料 → 领用登记 → 使用记录
```

### 知识库更新流程
```
知识提交 → 专家审核 → 发布上线 → 版本管理 → 定期更新
```

## 📈 关键指标 (KPI)

| 指标 | 公式 | 目标值 |
|------|------|--------|
| 工单响应及时率 | 及时响应工单数 / 总工单数 | ≥ 95% |
| 工单处理及时率 | 及时处理工单数 / 总工单数 | ≥ 90% |
| 一次解决率 | 一次解决工单数 / 总工单数 | ≥ 85% |
| 客户满意度 | 满意评价数 / 总评价数 | ≥ 95% |
| 配件库存准确率 | 库存准确配件数 / 总配件数 | ≥ 98% |
| 知识库使用率 | 有查阅工单数 / 总工单数 | ≥ 60% |

## 🔧 技术实现

### 数据表结构
```sql
-- 服务工单表
CREATE TABLE service_tickets (
  id UUID PRIMARY KEY,
  ticket_number VARCHAR(50),
  customer_id UUID,
  product_id UUID,
  type VARCHAR(50),
  priority VARCHAR(20),
  status VARCHAR(20),
  description TEXT,
  assigned_to UUID,
  created_at TIMESTAMP,
  resolved_at TIMESTAMP
);

-- 服务进度表
CREATE TABLE service_progress (
  id UUID PRIMARY KEY,
  ticket_id UUID,
  step VARCHAR(50),
  status VARCHAR(20),
  started_at TIMESTAMP,
  completed_at TIMESTAMP,
  notes TEXT
);

-- 服务评价表
CREATE TABLE service_reviews (
  id UUID PRIMARY KEY,
  ticket_id UUID,
  customer_id UUID,
  overall_rating INT,
  attitude_rating INT,
  skill_rating INT,
  speed_rating INT,
  comment TEXT,
  created_at TIMESTAMP
);

-- 配件库存表
CREATE TABLE service_parts_inventory (
  id UUID PRIMARY KEY,
  part_id UUID,
  warehouse_id UUID,
  quantity INT,
  min_quantity INT,
  max_quantity INT,
  updated_at TIMESTAMP
);

-- 知识库表
CREATE TABLE knowledge_base (
  id UUID PRIMARY KEY,
  title VARCHAR(200),
  category VARCHAR(50),
  tags TEXT[],
  content TEXT,
  version INT,
  is_published BOOLEAN,
  created_at TIMESTAMP,
  updated_at TIMESTAMP
);
```
