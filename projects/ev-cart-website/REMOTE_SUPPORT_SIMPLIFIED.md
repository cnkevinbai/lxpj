## 📊 功能评估

### 原设计方案（过于复杂）
```
远程指导 = 电话记录 + 视频指导 + 图文指导 + 屏幕共享

问题:
❌ 视频指导需要 WebRTC 技术，复杂度高
❌ 屏幕共享需要专用 SDK，成本高
❌ 实时通信需要额外服务器，运维成本高
❌ 功能过于先进，客户接受度低
```

### 简化设计方案（推荐）
```
远程指导 = 电话记录 + 图文指导

优势:
✅ 电话记录 - 简单实用，只需记录通话信息
✅ 图文指导 - 类似微信聊天，客户熟悉
✅ 实现简单 - 无需额外技术栈
✅ 成本低 - 无需额外服务器和 SDK
```

## 📋 简化版 API 设计

### 电话指导 API
```typescript
// 创建通话记录
POST /service/remote/calls
{
  ticketId: string
  customerPhone: string
  startTime: string
  endTime: string
  notes: string
}

// 获取通话记录
GET /service/tickets/:ticketId/remote/calls

// 删除通话记录
DELETE /service/remote/calls/:callId
```

### 图文指导 API
```typescript
// 发送消息
POST /service/remote/messages
{
  ticketId: string
  type: 'text' | 'image' | 'file'
  content: string
}

// 获取消息历史
GET /service/tickets/:ticketId/remote/messages

// 上传文件
POST /service/remote/upload
Content-Type: multipart/form-data

// 标记已读
POST /service/remote/messages/:messageId/read
```

### 完成指导 API
```typescript
// 完成远程指导
POST /service/tickets/:ticketId/remote/complete
{
  resolved: boolean
  resolutionNotes: string
  duration: number  // 分钟
  customerSatisfaction: number  // 1-5 星
}
```
