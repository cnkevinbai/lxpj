## 二、BUG 清单

### 🔴 高优先级 BUG (3 个)

#### BUG-20260313-001 🔴
**模块**: 产品详情页  
**严重程度**: 高  
**问题**: 产品详情页路由参数解析错误，导致产品 ID 为 undefined

**复现步骤**:
1. 访问产品中心页面
2. 点击任意产品"查看详情"按钮
3. 页面报错或显示空数据

**预期结果**: 显示产品详情
**实际结果**: 页面空白或报错

**根因分析**: 
- ProductDetail 组件 useParams 钩子使用正确
- 但路由配置中 products/:id 路径可能与其他路由冲突

**解决方案**: 
```javascript
// 修改 App.tsx 路由顺序，将动态路由放在静态路由之后
<Route path="products/compare" element={<ProductCompare />} />
<Route path="products/:id" element={<ProductDetail />} />
```

**修复状态**: ⏳ 待修复

---

#### BUG-20260313-002 🔴
**模块**: 服务工单提交  
**严重程度**: 高  
**问题**: 服务工单提交后未调用后端 API

**复现步骤**:
1. 访问服务支持页面
2. 填写服务工单表单
3. 点击提交按钮
4. 显示成功但后端未收到数据

**预期结果**: 数据提交到后端并创建工单
**实际结果**: 仅前端显示成功，后端无数据

**根因分析**: 
- 表单 onSubmit 未正确调用 API
- 缺少实际的 fetch 调用代码

**解决方案**: 
```javascript
const onFinish = async (values) => {
  try {
    const response = await fetch('/api/v1/service/tickets', {
      method: 'POST',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify(values),
    })
    const result = await response.json()
    if (result.success) {
      message.success('工单创建成功')
    }
  } catch (error) {
    message.error('提交失败')
  }
}
```

**修复状态**: ⏳ 待修复

#### BUG-20260313-003 🔴
**模块**: 经销商加盟申请  
**严重程度**: 高  
**问题**: 加盟申请表单字段与后端 API 不匹配

**复现步骤**:
1. 访问经销商加盟页面
2. 填写加盟申请表单
3. 提交申请
4. 后端返回字段错误

**预期结果**: 申请成功创建
**实际结果**: 后端返回字段验证错误

**根因分析**: 
- 前端表单字段名与后端 API 期望字段不一致
- 缺少字段映射

**解决方案**: 
```javascript
// 前端提交时进行字段映射
const submitApplication = async (formData) => {
  const apiData = {
    name: formData.name,
    phone: formData.phone,
    company: formData.company,
    city: formData.city,
    investment: formData.investment,
    experience: formData.experience,
    message: formData.message,
  }
  await fetch('/api/v1/dealers/apply', {
    method: 'POST',
    body: JSON.stringify(apiData),
  })
}
```

**修复状态**: ⏳ 待修复

  const stock = await response.json()

    // 库存不足，创建生产计划

}
