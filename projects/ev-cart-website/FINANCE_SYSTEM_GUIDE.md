## 📊 系统概览

道达智能财务系统覆盖企业财务管理全流程，包括预算、资金、核算、报表四大核心领域，实现财务业务一体化。

### 核心模块
| 模块 | 功能 | 状态 |
|------|------|------|
| 财务管理 | 应收应付、发票、费用报销 | ✅ |
| 成本管理 | 标准成本、实际成本、成本分析 | ✅ |
| 资产管理 | 固定资产、折旧、处置 | ✅ |
| 预算管理 | 预算编制、执行、预警 | ✅ |
| 资金管理 | 资金流水、现金流、资金调拨 | ✅ |
| 信用管理 | 客户授信、额度监控、风控 | ✅ |

#### 信用评估模型
```
信用评分 = 财务状况 (30%) + 历史付款 (40%) + 经营规模 (20%) + 行业风险 (10%)

信用等级:
AAA (90-100): 优秀，宽松政策
AA  (80-89):  良好，正常政策
A   (70-79):  一般，谨慎政策
BBB (60-69):  较差，严格政策
BB  (<60):    高风险，停止授信
```

## 🔄 核心业务流程

### 应收流程
```
销售出库 → 生成应收 → 开具发票 → 客户付款 → 回款核销 → 应收结清
```

### 应付流程
```
采购入库 → 生成应付 → 收到发票 → 付款申请 → 付款执行 → 应付核销
```

### 费用报销流程
```
员工申请 → 部门审批 → 财务审核 → 领导审批 → 出纳付款 → 报销完成
```

### 预算编制流程
```
预算通知 → 部门申报 → 财务汇总 → 预算审批 → 预算下达 → 执行控制
```

## 📈 关键指标 (KPI)

| 指标 | 公式 | 目标值 |
|------|------|--------|
| 应收账款周转率 | 销售收入 / 平均应收账款 | ≥ 6 次/年 |
| 应收账款周转天数 | 365 / 周转率 | ≤ 60 天 |
| 应付账款周转率 | 采购成本 / 平均应付账款 | ≥ 4 次/年 |
| 预算执行率 | 实际支出 / 预算支出 | 95-105% |
| 成本偏差率 | (实际成本 - 标准成本) / 标准成本 | ≤ 5% |
| 资金计划准确率 | 实际现金流 / 预测现金流 | ≥ 90% |
| 费用报销及时率 | 按时报销笔数 / 总笔数 | ≥ 95% |

## 🔧 技术实现

### 数据表结构
```sql
-- 应收账款
CREATE TABLE finance_receivables (
  id UUID PRIMARY KEY,
  customer_id UUID,
  order_id UUID,
  amount DECIMAL(18,2),
  paid_amount DECIMAL(18,2),
  due_date DATE,
  status VARCHAR(20),
  created_at TIMESTAMP
);

-- 应付账款
CREATE TABLE finance_payables (
  id UUID PRIMARY KEY,
  supplier_id UUID,
  purchase_order_id UUID,
  amount DECIMAL(18,2),
  paid_amount DECIMAL(18,2),
  due_date DATE,
  status VARCHAR(20),
  created_at TIMESTAMP
);

-- 资产卡片
CREATE TABLE erp_assets (
  id UUID PRIMARY KEY,
  asset_code VARCHAR(50),
  asset_name VARCHAR(200),
  asset_category VARCHAR(50),
  original_value DECIMAL(18,2),
  net_salvage_rate DECIMAL(5,2),
  depreciation_months INT,
  depreciation_method VARCHAR(50),
  department_id UUID,
  location VARCHAR(200),
  status VARCHAR(20),
  purchase_date DATE,
  in_service_date DATE
);

-- 预算计划
CREATE TABLE budget_plans (
  id UUID PRIMARY KEY,
  budget_year INT,
  department_id UUID,
  category VARCHAR(100),
  budget_amount DECIMAL(18,2),
  actual_amount DECIMAL(18,2),
  variance DECIMAL(18,2),
  variance_rate DECIMAL(5,2)
);

-- 信用额度
CREATE TABLE credit_limits (
  id UUID PRIMARY KEY,
  customer_id UUID,
  credit_rating VARCHAR(10),
  credit_limit DECIMAL(18,2),
  used_limit DECIMAL(18,2),
  available_limit DECIMAL(18,2),
  credit_term_days INT,
  effective_date DATE,
  expiry_date DATE
);
```
