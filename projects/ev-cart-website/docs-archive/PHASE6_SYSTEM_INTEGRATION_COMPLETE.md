## 🔄 外贸 CRM 合并方案

### 架构设计

**之前**:
```
CRM 系统（国内客户）
外贸系统（外贸客户）
→ 两个独立系统，数据割裂
```

**现在**:
```
统一 CRM 系统
├── 国内客户（customerType = 'domestic'）
└── 外贸客户（customerType = 'foreign'）
→ 一个系统，根据用户角色自动过滤
```

---

### 用户角色体系

**角色类型**:
```typescript
type RoleType = 
  | 'domestic'    // 国内业务员
  | 'foreign'     // 外贸业务员
  | 'admin'       // 管理员
  | 'manager'     // 经理
  | 'service';    // 客服
```

**权限控制**:
```typescript
// 国内业务员
- 只能查看/创建国内客户
- 只能查看国内商机
- 只能查看国内订单

// 外贸业务员
- 只能查看/创建外贸客户
- 只能查看外贸商机
- 只能查看外贸订单
- 可使用外贸特有功能（WhatsApp、多币种等）

// 管理员/经理
- 可查看所有客户
- 可查看所有数据
- 可进行数据统计分析
```

---

### 统一客户实体

**核心字段**:
```typescript
{
  // 基础信息
  customerCode: string,      // 客户编码（DC/FC 前缀）
  customerType: 'domestic' | 'foreign',  // 客户类型
  customerName: string,      // 客户名称
  customerNameLocal: string, // 本地语言名称（外贸用）
  
  // 联系信息
  contactPhone: string,
  contactWhatsapp: string,   // 外贸用
  contactWechat: string,
  contactEmail: string,
  
  // 地址信息
  country: string,
  timezone: string,          // 外贸用
  
  // 业务信息
  currency: 'CNY' | 'USD' | ...,  // 默认币种
  creditLimit: number,
  
  // 外贸专用
  incoterms: string,         // 贸易术语
  portOfLoading: string,     // 装运港
  bankName: string,          // 银行名称
  swiftCode: string,         // SWIFT 代码
  taxNo: string,             // 税号
}
```

---

### 登录流程

```
1. 用户登录
   ↓
2. 获取用户角色（domestic/foreign/admin）
   ↓
3. 根据角色加载对应界面
   ↓
4. 数据自动过滤（国内/外贸）
```

**界面自适应**:
```typescript
// 国内业务员界面
- 显示人民币（CNY）
- 显示国内省份
- 隐藏外贸功能

// 外贸业务员界面
- 显示多币种（USD/EUR 等）
- 显示国家/时区
- 显示外贸功能（WhatsApp、信用证等）
```

## 📊 数据互通

### 官网→CRM

**数据流**:
```
官网询价表单 → CRM 线索
官网联系表单 → CRM 联系人
官网产品浏览 → CRM 行为记录
```

**API**:
```typescript
// 官网询价
POST /api/v1/website/inquiry
→ 自动创建 CRM 线索
→ 分配给对应销售员

// 官网联系
POST /api/v1/website/contact
→ 自动创建 CRM 联系人
→ 发送确认邮件
```

---

### CRM→ERP

**数据流**:
```
CRM 客户 → ERP 客户档案
CRM 订单 → ERP 销售订单
CRM 商机 → ERP 销售预测
```

---

### ERP→APP

**数据流**:
```
ERP 订单 → APP 订单查询
ERP 库存 → APP 库存查询
ERP 生产 → APP 进度跟踪
```
