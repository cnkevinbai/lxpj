## 👥 业务员绑定逻辑

### 客户与业务员关系

| 关系类型 | 说明 |
|---------|------|
| 负责人 (owner) | 主要跟进的业务员 |
| 协作者 | 协助跟进的业务员 |
| 历史跟进人 | 曾经跟进过的业务员 |

### 绑定方式

1. **自动分配**
   - 线索转化为客户时自动绑定
   - 根据区域/行业自动分配

2. **手动分配**
   - 管理员手动分配
   - 业务员申请领取

3. **公海池领取**
   - 未分配客户放入公海
   - 业务员主动领取

## 🔌 API 接口

### 跟进记录

| 接口 | 方法 | 说明 |
|-----|------|------|
| `/follow-up` | POST | 创建跟进记录 |
| `/follow-up/target/:type/:id` | GET | 获取目标跟进记录 |
| `/follow-up/user/:userId` | GET | 获取业务员跟进记录 |
| `/follow-up/pending/:userId` | GET | 获取待跟进记录 |
| `/follow-up/:id` | PUT | 更新跟进记录 |
| `/follow-up/:id` | DELETE | 删除跟进记录 |
| `/follow-up/stats/:userId` | GET | 获取跟进统计 |
| `/follow-up/team-stats` | GET | 获取团队统计 |

### 客户分配

| 接口 | 方法 | 说明 |
|-----|------|------|
| `/customers/assign` | POST | 分配客户 |
| `/customers/batch-assign` | POST | 批量分配 |
| `/customers/by-owner/:ownerId` | GET | 获取业务员客户 |
| `/customers/unassigned` | GET | 获取未分配客户 |

## 📊 使用场景

### 场景 1：日常跟进

1. 业务员打开客户详情
2. 点击"添加跟进"
3. 选择跟进方式 (电话/拜访/微信)
4. 填写跟进内容
5. 设置下次跟进计划
6. 提交保存

### 场景 2：主管查看团队业绩

1. 主管打开业绩看板
2. 选择日期范围
3. 查看团队总数据
4. 查看业务员排名
5. 分析业绩趋势

### 场景 3：客户分配

1. 管理员进入客户管理
2. 选择未分配客户
3. 选择目标业务员
4. 批量分配
5. 通知业务员
