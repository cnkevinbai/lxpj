## 🔌 后端 API 服务器完成

### API 服务器 (`/api-server`) ✅

**技术栈**: Node.js + Express + MongoDB

**API 接口** (25+ 个):

#### 认证模块
- `POST /api/v1/auth/login` - 用户登录
- `POST /api/v1/auth/logout` - 用户登出

#### 客户管理
- `GET /api/v1/customers` - 客户列表
- `GET /api/v1/customers/:id` - 客户详情
- `POST /api/v1/customers` - 创建客户
- `PUT /api/v1/customers/:id` - 更新客户
- `DELETE /api/v1/customers/:id` - 删除客户

#### 销售管理
- `GET /api/v1/leads` - 线索列表
- `GET /api/v1/opportunities` - 商机列表
- `GET /api/v1/orders` - 订单列表

#### 产品管理
- `GET /api/v1/products` - 产品列表

#### 生产管理
- `GET /api/v1/production/orders` - 生产订单
- `POST /api/v1/production/orders` - 创建生产订单

#### 质量管理
- `GET /api/v1/quality/checks` - 质检记录
- `POST /api/v1/quality/checks` - 创建质检

#### 设备管理
- `GET /api/v1/equipment` - 设备列表
- `POST /api/v1/equipment` - 添加设备

#### 财务管理
- `GET /api/v1/finance/statistics` - 财务统计
- `GET /api/v1/finance/cashflow` - 资金流水

#### 预算管理
- `GET /api/v1/budget` - 预算列表
- `POST /api/v1/budget` - 创建预算

#### 人力资源
- `GET /api/v1/hr/employees` - 员工列表
- `GET /api/v1/hr/training` - 培训列表

#### 审批流
- `GET /api/v1/workflow/approvals` - 审批列表
- `POST /api/v1/workflow/approvals/:id/approve` - 审批通过
- `POST /api/v1/workflow/approvals/:id/reject` - 审批拒绝

#### 钉钉集成
- `GET /api/v1/dingtalk/status` - 钉钉状态
- `POST /api/v1/dingtalk/sync` - 同步数据

#### 报表中心
- `GET /api/v1/reports/sales` - 销售报表
- `GET /api/v1/reports/production` - 生产报表

#### 系统设置
- `GET /api/v1/settings/company` - 公司信息
- `PUT /api/v1/settings/company` - 更新公司信息

**启动方式**:
```bash
cd api-server
npm install
npm start
# 服务运行在 http://localhost:3000
```
