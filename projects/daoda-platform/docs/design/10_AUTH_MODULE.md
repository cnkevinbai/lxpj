## 一、模块概述

### 1.1 模块定位

权限系统是企业数字化平台的基础设施模块，负责用户身份认证、权限管理、组织架构、多租户隔离等核心功能，是所有业务模块的安全基石。

### 1.3 主流权限系统对比分析

#### 开源框架

| 系统 | 优势 | 劣势 | 借鉴点 |
|------|------|------|--------|
| **Keycloak** | • 功能全面<br>• 标准协议支持<br>• 社区活跃<br>• 可定制性强 | • 学习曲线陡峭<br>• 部署复杂 | • OAuth2/OIDC实现<br>• 多租户设计 |

| **Apache Shiro** | • 轻量级<br>• Java生态 | • 功能相对简单 | • 权限模型设计 |
| **Spring Security** | • Spring生态<br>• 功能强大 | • 配置复杂 | • 安全过滤器链 |

#### 商业产品

| 产品 | 优势 | 劣势 | 借鉴点 |

## 二、权限模型设计

### 2.1 RBAC权限模型

```
┌─────────────────────────────────────────────────────────────────┐
│                     RBAC 权限模型                                │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   ┌─────────┐         ┌─────────┐         ┌─────────┐         │
│   │  用户   │ ──────▶ │  角色   │ ──────▶ │  权限   │         │
│   │  User   │  N:N    │  Role   │  N:N    │ Permission│        │
│   └─────────┘         └─────────┘         └─────────┘         │
│        │                   │                                    │
│        │                   │                                    │
│        ▼                   ▼                                    │
│   ┌─────────┐         ┌─────────┐                              │
│   │ 组织    │         │ 菜单    │                              │
│   │ Org     │         │ Menu    │                              │
│   └─────────┘         └─────────┘                              │
│                                                                 │
│  【核心概念】                                                    │
│                                                                 │
│  用户(User)                                                      │
│  ├─ 系统使用者，可属于多个组织                                  │
│  └─ 可分配多个角色                                              │
│                                                                 │
│  角色(Role)                                                      │
│  ├─ 权限的集合，代表岗位职责                                    │
│  ├─ 可继承其他角色                                              │
│  └─ 可分配给多个用户                                            │
│                                                                 │
│  权限(Permission)                                                │
│  ├─ 最小权限单元                                                │
│  ├─ 格式: module:resource:action                               │
│  └─ 示例: crm:customer:create, erp:order:view                  │
│                                                                 │
│  菜单(Menu)                                                      │
│  ├─ 系统导航菜单                                                │
│  ├─ 与权限关联                                                  │
│  └─ 控制菜单可见性                                              │
│                                                                 │
│  组织(Organization)                                              │
│  ├─ 多级组织架构                                                │
│  ├─ 用户所属组织                                                │
│  └─ 数据权限范围                                                │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 2.2 数据权限模型

```
┌─────────────────────────────────────────────────────────────────┐
│                     数据权限模型                                 │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  【数据权限范围】                                                │
│                                                                 │
│  1️⃣ 全部数据                                                    │
│     • 可访问系统中所有数据                                      │
│     • 适用于超级管理员                                          │
│                                                                 │
│  2️⃣ 本组织及下级数据                                            │
│     • 可访问本组织及其所有下级组织的数据                        │
│     • 适用于组织负责人                                          │
│                                                                 │
│  3️⃣ 本组织数据                                                  │
│     • 只能访问本组织的数据                                      │
│     • 适用于部门员工                                            │
│                                                                 │
│  4️⃣ 本人数据                                                    │
│     • 只能访问自己创建/负责的数据                               │
│     • 适用于普通员工                                            │
│                                                                 │
│  5️⃣ 自定义数据                                                  │
│     • 按自定义规则过滤                                          │
│     • 支持SQL条件表达式                                         │
│                                                                 │
│  【数据权限配置】                                                │
│                                                                 │
│  ┌─────────────┐                                               │
│  │ 数据权限规则 │                                               │
│  ├─────────────┤                                               │
│  │ 规则名称    │ 销售部数据权限                                │
│  │ 适用模块    │ CRM, ERP                                      │
│  │ 数据范围    │ 本组织及下级                                  │
│  │ 适用表      │ customers, orders, contracts                  │
│  │ 条件表达式  │ department_id IN (SELECT id FROM department   │
│  │             │ WHERE path LIKE '${currentOrgPath}%')         │
│  └─────────────┘                                               │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 2.3 多租户模型

```
┌─────────────────────────────────────────────────────────────────┐
│                     多租户模型                                   │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  【租户隔离模式】                                                │
│                                                                 │
│  模式1: 独立数据库 (Database per Tenant)                        │
│  ┌─────────┐  ┌─────────┐  ┌─────────┐                         │
│  │ 租户A   │  │ 租户B   │  │ 租户C   │                         │
│  │  DB-A   │  │  DB-B   │  │  DB-C   │                         │
│  └─────────┘  └─────────┘  └─────────┘                         │
│  优点: 隔离性强, 备份恢复方便                                   │
│  缺点: 成本高, 维护复杂                                         │
│  适用: 大型企业租户                                             │
│                                                                 │
│  模式2: 共享数据库, 独立Schema (Schema per Tenant)              │
│  ┌─────────────────────────────────────────────┐               │
│  │              共享数据库                      │               │
│  │  ┌─────────┐  ┌─────────┐  ┌─────────┐    │               │
│  │  │Schema A │  │Schema B │  │Schema C │    │               │
│  │  │  租户A  │  │  租户B  │  │  租户C  │    │               │
│  │  └─────────┘  └─────────┘  └─────────┘    │               │
│  └─────────────────────────────────────────────┘               │
│  优点: 隔离性较好, 成本适中                                     │
│  缺点: 跨租户查询复杂                                           │
│  适用: 中型租户                                                 │
│                                                                 │
│  模式3: 共享数据库, 共享Schema (Shared Schema) ← 推荐           │
│  ┌─────────────────────────────────────────────┐               │
│  │              共享数据库                      │               │
│  │  ┌───────────────────────────────────────┐ │               │
│  │  │           共享Schema                   │ │               │
│  │  │  表数据通过 tenant_id 字段区分         │ │               │
│  │  └───────────────────────────────────────┘ │               │
│  └─────────────────────────────────────────────┘               │
│  优点: 成本低, 维护简单                                         │
│  缺点: 隔离性弱, 需要应用层隔离                                 │
│  适用: 中小型租户                                               │
│                                                                 │
│  【道达智能选择】                                                │
│  • 采用模式3: 共享Schema, 通过tenant_id隔离                     │
│  • 配合数据权限实现细粒度隔离                                   │
│  • 支持租户自定义配置                                           │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

## 六、权限控制实现

### 6.1 权限守卫

```typescript
// permission.guard.ts

import {
  Injectable,
  CanActivate,
  ExecutionContext,
  ForbiddenException,
} from '@nestjs/common';
import { Reflector } from '@nestjs/core';
import { PERMISSION_KEY } from './permission.decorator';

@Injectable()
export class PermissionGuard implements CanActivate {
  constructor(private reflector: Reflector) {}

  canActivate(context: ExecutionContext): boolean {
    // 获取所需权限
    const requiredPermissions = this.reflector.getAllAndOverride<string[]>(
      PERMISSION_KEY,
      [context.getHandler(), context.getClass()],
    );

    if (!requiredPermissions || requiredPermissions.length === 0) {
      return true;
    }

    const request = context.switchToHttp().getRequest();
    const user = request.user;

    if (!user) {
      throw new ForbiddenException('未登录');
    }

    // 超级管理员跳过权限检查
    if (user.isSuperAdmin) {
      return true;
    }

    // 检查权限
    const userPermissions = user.permissions || [];
    const hasPermission = requiredPermissions.some((permission) =>
      userPermissions.includes(permission),
    );

    if (!hasPermission) {
      throw new ForbiddenException('无权限访问');
    }

    return true;
  }
}
```

  filter(query: any, user: User): any;
}

/**
 * 用户事件监听扩展接口
 * 允许模块监听用户相关事件
 */
export interface IUserEventListener {
  onUserCreated(user: User): Promise<void>;
  onUserUpdated(user: User): Promise<void>;
  onUserDeleted(userId: string): Promise<void>;
  onUserLogin(user: User, context: LoginContext): Promise<void>;
  onUserLogout(userId: string): Promise<void>;
}
