## 1. 项目概述

### 1.1 项目背景

四川道达智能车辆制造有限公司专注于无人驾驶电动观光车和常规新能源电动观光车的研发与制造。随着智能网联汽车技术的发展，建设一套完整的车联网管理平台成为企业数字化转型的关键基础设施。

### 1.2 项目目标

构建一套**高可用、高可靠、高扩展、数据安全合规**的企业级车联网车辆管理平台，实现：

- **全生命周期管理**: 车辆从生产、销售、运营到报废的全生命周期数字化管理
- **实时监控**: 车辆位置、状态、故障的实时监控与预警
- **远程控制**: 支持远程诊断、远程升级、远程控制等功能
- **数据分析**: 车辆运行数据采集、存储、分析与可视化
- **智能调度**: 基于AI的车辆智能调度与路径规划
- **安全合规**: 满足国家车联网安全法规要求

### 1.3 适用场景

| 场景类型 | 具体应用 | 核心需求 |
|---------|---------|---------|
| 无人驾驶观光车 | 景区、园区、机场摆渡 | 高精度定位、自动避障、远程接管 |
| 新能源观光车 | 旅游景区、主题公园 | 电量监控、充电管理、运营调度 |
| 车队管理 | 企业车队、租赁车队 | 车辆调度、驾驶员管理、运营分析 |

## 2. 需求分析

### 2.1 功能需求

#### 2.1.1 车辆接入管理
- 支持多种通信协议接入（MQTT、TCP、HTTP、WebSocket）
- 支持多品牌、多车型车辆接入
- 支持车辆认证、鉴权、注册
- 支持车辆在线状态管理

#### 2.1.2 实时监控
- 车辆位置实时追踪（GPS/北斗）
- 车辆状态实时监控（速度、电量、里程等）
- 车辆告警实时推送
- 视频监控（可选）

#### 2.1.3 远程控制
- 远程锁车/解锁
- 远程诊断
- 远程OTA升级
- 远程参数配置

#### 2.1.4 数据管理
- 车辆档案管理
- 运营数据统计
- 故障数据分析
- 行驶轨迹回放

#### 2.1.5 运营管理
- 车辆调度
- 线路规划
- 电子围栏
- 运营报表

### 2.2 非功能需求

#### 2.2.1 初期目标（企业自用阶段）

| 指标类别 | 具体指标 | 目标值 | 说明 |
|---------|---------|--------|------|
| 可用性 | 系统可用率 | ≥99.5% | 单机房部署 |
| 可靠性 | 数据传输成功率 | ≥99.9% | 主备链路保障 |
| 性能 | 并发车辆接入 | ≥500辆 | 初期规模 |
| 性能 | 消息处理延迟 | <200ms | 满足业务需求 |
| 扩展性 | 水平扩展能力 | 支持动态扩缩容 | 架构预留扩展点 |
| 安全性 | 数据加密 | 国密算法 | SM2/SM3/SM4 |
| 合规性 | 数据存储 | 符合法规要求 | 数据分级存储 |

#### 2.2.2 扩展目标（商用阶段）

| 性能 | 并发车辆接入 | ≥10万辆 | 集群扩展 |
| 性能 | 消息处理延迟 | <100ms | 性能优化 |

/**
 * 车辆接入模块入口类
 * 
 * 模块入口类必须:
 * 1. 实现 IModule 接口
 * 2. 添加 @Module 注解
 * 3. 提供无参构造函数
 */
@Module(
    id = "vehicle-access",
    name = "车辆接入服务",
    version = "1.0.0",
    description = "负责车辆连接认证、会话管理",
    dependencies = {
        @Dependency(name = "common-core", version = ">=1.0.0"),
        @Dependency(name = "common-mqtt", version = ">=1.0.0", optional = true)
    },
    extensionPoints = {
        @ExtensionPoint(name = "vehicle-auth-handler", 
                        interfaceClass = VehicleAuthHandler.class)
    }
)
public class VehicleAccessModule implements IHotReloadableModule {
    
    private ModuleContext context;
    private ModuleState state = ModuleState.CREATED;
    private VehicleAccessConfig config;
    
    @Override
    public ModuleMetadata getMetadata() {
        return ModuleMetadata.builder()
            .id("vehicle-access")
            .name("车辆接入服务")
            .version("1.0.0")
            .build();
    }
    
    @Override
    public void initialize(ModuleContext context) throws ModuleException {
        this.context = context;
        this.state = ModuleState.INITIALIZING;
        
        try {
            this.config = context.getConfig(VehicleAccessConfig.class);
            validateConfig(config);
            
            this.state = ModuleState.INITIALIZED;
            log.info("VehicleAccessModule initialized successfully");
            
        } catch (Exception e) {
            this.state = ModuleState.FAILED;
            throw new ModuleException("Module initialization failed", e);
        }
    }
    
    @Override
    public void start() throws ModuleException {
        this.state = ModuleState.STARTING;
        
        try {
            registerServices();
            registerEventListeners();
            startAcceptors();
            
            this.state = ModuleState.RUNNING;
            log.info("VehicleAccessModule started successfully");
            
        } catch (Exception e) {
            this.state = ModuleState.FAILED;
            throw new ModuleException("Module start failed", e);
        }
    }
    
    @Override
    public void stop() throws ModuleException {
        this.state = ModuleState.STOPPING;
        
        try {
            stopAcceptors();
            unregisterEventListeners();
            unregisterServices();
            
            this.state = ModuleState.STOPPED;
            log.info("VehicleAccessModule stopped successfully");
            
        } catch (Exception e) {
            this.state = ModuleState.FAILED;
            throw new ModuleException("Module stop failed", e);
        }
    }
    
    @Override
    public void destroy() throws ModuleException {
        this.state = ModuleState.DESTROYED;
        this.context = null;
        log.info("VehicleAccessModule destroyed");
    }
    
    @Override
    public void updateConfig(ModuleConfig config) throws ModuleException {
        VehicleAccessConfig newConfig = (VehicleAccessConfig) config;
        validateConfig(newConfig);
        
        VehicleAccessConfig oldConfig = this.config;
        this.config = newConfig;
        
        try {
            applyConfigChanges(oldConfig, newConfig);
            log.info("Config updated successfully");
        } catch (Exception e) {
            this.config = oldConfig;
            throw new ModuleException("Config update failed, rolled back", e);
        }
    }
    
    @Override
    public ModuleState getState() {
        return this.state;
    }
    
    @Override
    public HealthStatus getHealthStatus() {
        return HealthStatus.builder()
            .status(state == ModuleState.RUNNING ? HealthStatus.UP : HealthStatus.DOWN)
            .details(Map.of(
                "connections", getCurrentConnections(),
                "maxConnections", config.getMaxConnections(),
                "uptime", getUptime()
            ))
            .build();
    }
}

#### 4.1.4 链路切换策略

```java
public class LinkSwitchStrategy {
    
    // 切换条件配置
    private static final int MAX_RETRY_COUNT = 3;
    private static final long HEARTBEAT_TIMEOUT = 30000; // 30秒
    private static final double LATENCY_THRESHOLD = 500.0; // 500ms
    private static final int SIGNAL_STRENGTH_THRESHOLD = -90; // dBm
    
    public LinkType determineBestLink(LinkStatus mqttStatus, LinkStatus tcpStatus) {
        // 评分机制
        double mqttScore = calculateLinkScore(mqttStatus);
        double tcpScore = calculateLinkScore(tcpStatus);
        
        // MQTT优先策略
        if (mqttScore >= 80) {
            return LinkType.MQTT;
        } else if (tcpScore >= 80) {
            return LinkType.TCP;
        } else if (mqttScore >= tcpScore) {
            return LinkType.MQTT;
        } else {
            return LinkType.TCP;
        }
    }
    
    private double calculateLinkScore(LinkStatus status) {
        double score = 100.0;
        
        // 延迟惩罚
        if (status.getLatency() > LATENCY_THRESHOLD) {
            score -= (status.getLatency() - LATENCY_THRESHOLD) / 10;
        }
        
        // 丢包率惩罚
        score -= status.getPacketLoss() * 50;
        
        // 信号强度惩罚
        if (status.getSignalStrength() < SIGNAL_STRENGTH_THRESHOLD) {
            score -= (SIGNAL_STRENGTH_THRESHOLD - status.getSignalStrength()) * 0.5;
        }
        
        // 心跳超时惩罚
        if (status.getLastHeartbeat() > HEARTBEAT_TIMEOUT) {
            score -= 30;
        }
        
        return Math.max(0, score);
    }
}
```

SELECT

扩容触发条件:
- 车辆接入数超过当前容量80%
- CPU使用率持续超过70%
- 内存使用率持续超过80%
- 存储使用率超过70%
- 响应时间超过200ms

阶段升级建议:
- 阶段零 → 阶段一: 开发完成，准备上线运营
- 阶段一 → 阶段二: 车辆数超过400辆，或业务增长需要
- 阶段二 → 阶段三: 车辆数超过1500辆，或需要更高可用性
- 阶段三 → 阶段四: 车辆数超过4000辆，或需要异地容灾
