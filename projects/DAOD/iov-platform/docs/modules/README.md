# 车联网管理平台模块开发文档索引

## 文档概述

本文档提供车联网管理平台所有功能模块的开发文档索引，帮助开发人员快速定位和理解各模块的设计与实现。

## 模块分类

### 核心框架模块 (Core)

核心框架模块提供平台的基础架构能力，包括模块化框架、配置中心、认证服务等。

| 模块名称 | 文档路径 | 描述 | 优先级 |
|---------|---------|------|--------|
| 热更新引擎 | [hot-reload-engine.md](./core/hot-reload-engine.md) | 模块热插拔、热更新、热修改核心引擎 | P0 |
| 配置中心 | [config-center.md](./core/config-center.md) | 集中配置管理、热更新配置 | P0 |
| 认证服务 | [auth-service.md](./core/auth-service.md) | 统一身份认证、JWT令牌管理 | P0 |

### 多租户账号体系模块 (Tenant)

多租户账号体系模块支持生产厂家、经销商、运营者、景区管理员等多角色独立使用。

| 模块名称 | 文档路径 | 描述 | 优先级 |
|---------|---------|------|--------|
| 租户服务 | [tenant-service.md](./tenant/tenant-service.md) | 租户管理、租户隔离 | P0 |
| 子账号服务 | [sub-account-service.md](./tenant/sub-account-service.md) | 子账号管理、账号类型识别 | P0 |
| 角色权限服务 | [role-service.md](./tenant/role-service.md) | RBAC权限模型、角色定义 | P0 |

### 业务功能模块 (Business)

业务功能模块实现车联网平台的核心业务能力。

| 模块名称 | 文档路径 | 描述 | 优先级 |

| 模块名称 | 文档路径 | 描述 | 优先级 |

## 开发规范

### 模块开发流程

1. **需求分析**: 阅读模块开发文档，理解模块功能和接口设计
2. **接口定义**: 根据文档中的接口设计，实现模块接口
3. **数据模型**: 根据数据库设计，创建数据表和实体类
4. **业务实现**: 实现核心业务逻辑
5. **单元测试**: 编写单元测试用例
6. **集成测试**: 进行模块间集成测试
7. **文档更新**: 更新模块文档和API文档

### 模块描述文件

每个模块需要包含 `module.yaml` 描述文件：

```yaml
name: module-name
version: 1.0.0
type: business
priority: 60
description: 模块描述
author: 开发者
dependencies:
  - name: plugin-framework
    version: ">=1.0.0"
resources:
  cpu: "100m"
  memory: "128Mi"
```

### 代码规范

- 遵循Java编码规范
- 使用Spring Boot注解
- 统一异常处理
- 统一日志格式
- 单元测试覆盖率 >= 80%
