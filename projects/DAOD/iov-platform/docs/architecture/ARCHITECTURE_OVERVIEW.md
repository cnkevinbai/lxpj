## 1. 架构概述

### 1.1 设计理念

本系统采用 **"微内核 + 动态插件"** 的先进架构范式，彻底重构了传统单体应用的升级与维护模式。

核心设计理念遵循 **"高内聚、低耦合"** 原则，将业务功能封装为独立的 **标准化功能单元 (SFU)**。

### 1.2 核心特性

| 特性 | 描述 |
|------|------|
| **模块化** | 业务功能封装为独立的标准化功能单元 (SFU) |
| **热插拔** | 新功能模块可像 USB 设备一样在系统运行期间即时接入并生效 |
| **无感热更新** | 故障模块可被无缝隔离与替换，全程零停机、零中断、零感知 |
| **标准化** | 统一的模块接口定义、API 规范、监控指标 |
| **可扩展** | 系统具备无限的水平扩展能力 |

## 2. 微内核架构

### 2.1 核心组件

| 组件 | 职责 | 状态 |
|------|------|------|
| **PluginManager** | 模块加载、卸载、版本管理、依赖检查 | ✅ 已实现 |
| **SandboxManager** | 模块权限控制、资源配额、安全隔离 | ⚠️ 部分实现 |
| **CircuitBreaker** | 错误率监控、自动熔断、故障隔离 | ⚠️ 部分实现 |
| **RollbackManager** | 自动备份、失败回滚、版本切换 | ✅ 已实现 |
| **EventBus** | 模块间异步通信、事件发布订阅 | ✅ 已实现 |
| **ConfigCenter** | 配置管理、热更新推送、多环境隔离 | ✅ 已实现 |
| **HealthChecker** | 健康检查、存活探针、就绪探针 | ✅ 已实现 |
| **MetricsRegistry** | 指标注册、Prometheus 导出 | ⚠️ 部分实现 |

### 2.2 模块生命周期

### 2.3 热更新流程

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                           无感热更新流程                                     │
└─────────────────────────────────────────────────────────────────────────────┘

1. 备份当前版本
   ┌──────────┐
   │ 备份模块 │ ──▶ 创建备份点 (backup-v1.0.0)
   └──────────┘

2. 加载新版本
   ┌──────────┐     ┌──────────┐     ┌──────────┐
   │ 加载模块 │ ──▶ │ 检查依赖 │ ──▶ │ 初始化   │
   └──────────┘     └──────────┘     └──────────┘

3. 灰度切换 (策略: rolling/blue-green/canary)
   ┌──────────────────────────────────────────────────────────────────────┐
   │ Rolling Update:                                                      │
   │ ┌────────┐ ┌────────┐ ┌────────┐ ┌────────┐ ┌────────┐             │
   │ │ v1.0.0 │ │ v1.0.0 │ │ v1.1.0 │ │ v1.0.0 │ │ v1.1.0 │  ...        │
   │ └────────┘ └────────┘ └────────┘ └────────┘ └────────┘             │
   │   实例1     实例2     实例1'    实例3     实例2'                    │
   └──────────────────────────────────────────────────────────────────────┘

4. 健康检查
   ┌──────────┐     ┌──────────┐
   │ 健康检查 │ ──▶ │ 熔断检测 │
   └──────────┘     └──────────┘
        │                │
        ▼                ▼
   ┌──────────┐     ┌──────────┐
   │ 检查通过 │     │ 错误率过高 │
   │ 更新成功 │     │ 触发回滚   │
   └──────────┘     └──────────┘

5. 自动回滚 (如果健康检查失败)
   ┌──────────┐     ┌──────────┐     ┌──────────┐
   │ 熔断触发 │ ──▶ │ 隔离模块 │ ──▶ │ 回滚版本 │
   └──────────┘     └──────────┘     └──────────┘
```

## 3. 标准化功能单元 (SFU)

### 3.1 SFU 接口定义

```java
public interface ISFU extends IModule {
    
    // ==================== 元数据 ====================
    ModuleMetadata getMetadata();
    
    // ==================== 生命周期 ====================
    void initialize(ModuleContext context) throws ModuleException;
    void start() throws ModuleException;
    void stop() throws ModuleException;
    void destroy() throws ModuleException;
    
    // ==================== 状态查询 ====================
    ModuleState getState();
    HealthStatus getHealthStatus();
    
    // ==================== 监控指标 ====================
    List<Metric> getMetrics();
    HealthCheckResult healthCheck();
    
    // ==================== API 规范 ====================
    String getApiSpecification();
    List<ApiDependency> getApiDependencies();
    
    // ==================== 沙箱配置 ====================
    List<Permission> getRequiredPermissions();
    ResourceRequirements getResourceRequirements();
}
```

### 3.2 模块分类

| 类型 | 职责范围 | 加载顺序 | 示例 |
|------|---------|---------|------|
| **core** | 核心框架能力 | 1 | plugin-framework, hot-reload-engine |
| **adapter** | 协议适配 | 2 | jtt808-adapter, mqtt-adapter |
| **business** | 业务功能 | 3 | vehicle-access, monitor-service |
| **extension** | 功能扩展 | 4 | custom-report, third-party-integration |
