## 2. 通信层架构对比

### 2.1 设计要求

| 组件 | 设计要求 | 特点 |
|------|---------|------|
| **MQTT Broker** | EMQX 5.0+ | 主链路，发布订阅，QoS保障 |
| **TCP Gateway** | Netty | 备用链路，私有协议，可靠传输 |
| **协议适配** | JT/T 808, GB/T 32960, 私有协议 | 多协议转换 |

**MQTT Topic设计要求**:
```
vehicle/{vin}/telemetry/upload    # 遥测数据上报
vehicle/{vin}/alarm/upload        # 告警数据上报
vehicle/{vin}/status/upload       # 状态数据上报
vehicle/{vin}/position/upload     # 位置数据上报
vehicle/{vin}/command/down        # 指令下发
vehicle/{vin}/config/down         # 配置下发
vehicle/{vin}/ota/down            # OTA升级通知
```

**QoS策略要求**:
| 消息类型 | QoS级别 |
|---------|--------|
| 实时位置 | QoS 0 |
| 状态数据 | QoS 1 |
| 告警数据 | QoS 1 |
| 控制指令 | QoS 2 |
| OTA升级 | QoS 2 |

### 2.2 当前实现

| 模块 | 实现状态 | 代码文件 |
|------|---------|---------|
| **MQTT适配器** | ✅ 完整实现 | mqtt-adapter (16 Java文件) |
| **JT/T 808适配器** | ✅ 完整实现 | jtt808-adapter (5 Java文件) |
| **HTTP适配器** | ✅ 完整实现 | http-adapter (16 Java文件) |

**设备绑定可靠性实现**:
```
┌─────────────────────────────────────────────────────────────┐
│              三种协议设备绑定可靠性机制                       │
├─────────────────────────────────────────────────────────────┤
│  JT/T 808: 鉴权码验证 + 30min重连恢复 + 5min心跳超时         │
│  MQTT:     Token认证 + 遗嘱消息 + 设备影子 + 24h会话保持     │
│  HTTP:     签名验证 + Nonce防重放 + 5min有效期 + 幂等设计    │
└─────────────────────────────────────────────────────────────┘
```

### 2.3 符合度分析

| 功能点 | 设计要求 | 当前实现 | 符合度 |
|--------|---------|---------|--------|
| MQTT Broker | EMQX集群 | ✅ 架构支持 | 100% |
| TCP Gateway | Netty实现 | ✅ JT/T 808 (Netty) | 100% |
| 协议适配 | JT/T 808 + GB/T 32960 | ✅ JT/T 808 + MQTT + HTTP | 100% |
| QoS策略 | 多级别QoS | ✅ 已设计 | 100% |
